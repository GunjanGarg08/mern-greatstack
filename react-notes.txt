In React, components are the building blocks of the UI. They can be created in two main ways:

1. Functional Components
2. Class-based Components
With the introduction of React Hooks (React 16.8+), functional components can now handle state and lifecycle logic, making them the preferred choice in modern React development.

1. Functional Components
Definition: JavaScript functions that return JSX.
State(useState) & Lifecycle(useEffect): Managed using Hooks (useState, useEffect, etc.).
Syntax: Simpler and more concise.
Performance: Slightly better due to less overhead.
Best for: Most modern React apps.


Example:

import React, { useState, useEffect } from 'react';

function Counter() {
  const [count, setCount] = useState(0); // State hook

  useEffect(() => {
    console.log(`Count updated: ${count}`);
  }, [count]); // Effect hook

  return (
    <div>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;


2. Class-based Components
Definition: ES6 classes that extend React.Component.
State & Lifecycle: Managed using this.state and lifecycle methods (componentDidMount, componentDidUpdate, componentWillUnmount, etc.).
Syntax: More verbose.
Performance: Slightly more overhead.
Best for: Legacy codebases or when working with older React patterns.


Example:

import React, { Component } from 'react';

class Counter extends Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 }; // State initialization
  }

  componentDidUpdate() {
    console.log(`Count updated: ${this.state.count}`);
  }

  render() {
    return (
      <div>
        <h2>Count: {this.state.count}</h2>
        <button onClick={() => this.setState({ count: this.state.count + 1 })}>
          Increment
        </button>
      </div>
    );
  }
}

export default Counter;

Use functional components with Hooks for new projects — they are cleaner, easier to test, and align with modern React best practices. Class components are still supported but mainly for maintaining older code.

In React development with VS Code,
rafc and rafce are code snippets provided by the ES7+ React/Redux/React-Native snippets extension.
They are shortcuts to quickly generate boilerplate code for functional components.


Meaning of rafc and rafce:
rafc	React Arrow Function Component	        Creates a functional component with named export.
rafce	React Arrow Function Component Export	Creates a functional component with default export.

Example: rafc
Typing rafc + Tab (or Enter) generates:

import React from 'react';

export const MyComponent = () => {
  return (
    <div>MyComponent</div>
  );
};

Named export — you must import it with { MyComponent }.


Example: rafce
Typing rafce + Tab generates:

import React from 'react';

const MyComponent = () => {
  return (
    <div>MyComponent</div>
  );
};

export default MyComponent;

Default export — you can import it without curly braces.


How to Enable These Shortcuts
1. Install the extension:
    1. ES7+ React/Redux/React-Native snippets by dsznajder.
2. In VS Code:
    1. Open Extensions (Ctrl+Shift+X / Cmd+Shift+X on Mac).
    2. Search for ES7+ React/Redux/React-Native snippets and install.
3. Make sure your file has:
.js, .jsx, .ts, or .tsx extension.
Language mode set to JavaScript React or TypeScript React (check bottom-right corner of VS Code).
Type rafc or rafce and press Tab (or Enter).

Tip:
1. Use rafc when you want multiple named components in one file.
2. Use rafce when the file will export only one main component.

import { useState } from "react";
import FirstComponent from "./components/FirstComponent";
const App = () => {

    // let x = 0;

    const [x, setX] = useState(0);
    // const [variable_name, function_name] = useState(initial_value_of_variable)

    const btnClick = () => {
        console.log("Clicked");
        // x++;
        setX(x+1); // so now using this function we need to update the value of x
        // the interface will also be updated and we will see the changed value of x on webpage as well
        console.log(x);
        // but the increased value of 'x' is not shown on browser webpage, it is shown only in console...
        // this is because the react interface change only when the state is changing... (VirtualDOM - asked in FSE viva by Sir)
        // hence for this we will use "useState" hook...
    }

    return (
        <div>
            // {x}
            <button onClick={() => {btnClick()}}>Click Me</button>
            <FirstComponent data={x} fn={setX}/>
            // props(variable) - <component_name prop_name={variable_name}/>
            // props(function) - <component_name prop_name={function_name}/>
        </div>
    )
}

export default App

Using "useRef" to get any element -> just like we use document.getElementByID(id_name) in JavaScript

App.jsx:

import { useRef } from "react"

const App = () => {
    
    const [data, setData] = useState([]);
    const inputRef = useRef(null);

    return (
        <div>
        <input ref={inputRef} type="text" />
        // <button onClick={() => {console.log(inputRef.current.value);}}>Submit</button>
        <button onClick={() => {setData([...data, inputRef.current.value])}}>Submit</button>
        {data.map((item, index) => {return <h2 key={index}>{item}</h2>})};
        </div>
    )
}

export default App
